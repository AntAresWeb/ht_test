# API Организационной Структуры (Python + FastAPI)

**Проект:** Реализация REST API для управления иерархической структурой организации (подразделения и сотрудники) в соответствии с принципами чистой архитектуры.

**Технологии:**
*   **Backend:** Python 3.14, FastAPI, SQLAlchemy (Async), Pydantic.
*   **База данных:** PostgreSQL.
*   **Инфраструктура:** Docker, Docker Compose, Alembic (миграции).

---

## 📝 О Проекте

Это учебный/демонстрационный проект, созданный для отработки навыков построения сложных, масштабируемых и поддерживаемых приложений на Python.

**Ключевые архитектурные решения, которые демонстрирует проект:**

3.  **Unit of Work (UoW):** Управление транзакциями базы данных.
4.  **Обработка Ошибок:** Глобальные обработчики исключений в FastAPI.
5.  **Асинхронность:** Полностью асинхронный стек для высокой производительности.

## 🚀 Как запустить проект

Весь проект упакован в Docker для простоты запуска и изоляции зависимостей.

#### 0. Настройка окружения (Обязательно!)

В корне проекта находится файл **`.env.example`**. Это шаблон для конфигурации.
1.  Скопируйте его и переименуйте в **`.env`**:
    ```bash
    cp .env.example .env
    ```
2.  Откройте файл `.env` и задайте свои секреты для доступа к базе данных:
    ```bash
    # Найдите эти строки и измените значения
    POSTGRES_USER=your_secret_user
    POSTGRES_PASSWORD=your_strong_password
    # Остальные настройки (PORT, HOST) можно оставить по умолчанию
    ```

#### 1. Начальный запуск (Скачивание образов, создание БД и миграция)

Для первого запуска выполните одну команду. Она скачает образы, создаст контейнеры, инициализирует базу данных в папке `./database` и применит все миграции.

```bash
docker compose up --build -d
```
*   `--build` — пересобирает образ приложения (если код или Dockerfile изменились).
*   `-d` — запускает контейнеры в фоновом режиме (detached).

#### 2. Последующие запуски

Если вы остановили контейнеры, для их запуска используйте команду без пересборки образа:
```bash
docker compose up -d
```

#### 3. Остановка контейнеров

Чтобы остановить и удалить контейнеры (но сохранить данные в базе), используйте:
```bash
docker compose down
```

#### 4. Проверка работы API

Документация Swagger UI будет доступна по адресу:
[http://localhost:8000/docs](http://localhost:8000/docs)
*(Порт `8000` берется из переменной `.env`).*

## 📋 Доступные Эндпоинты

Документация доступна по адресу [http://localhost:8000/docs](http://localhost:8000/docs).
Ниже приведены основные эндпоинты и примеры использования через `curl`.
*(Не забудьте заменить `http://localhost:8000` на свой адрес и `1` на реальный ID).*

### Подразделения (Departments)
*   **Создать подразделение**
    ```bash
    curl -X POST "http://localhost:8000/departments/" -H "accept: application/json" -H "Content-Type: application/json" -d "{ \"name\": \"IT\", \"parent_id\": null }"
    ```
*   **Получить подразделение с деревом**
    ```bash
    curl -X GET "http://localhost:8000/departments/1?depth=2" -H "accept: application/json"
    ```
*   **Удалить подразделение (Каскад)**
    ```bash
    curl -X DELETE "http://localhost:8000/departments/1?mode=cascade" -H "accept: application/json"
    ```
*   **Удалить подразделение (Перенос)**
    ```bash
    curl -X DELETE "http://localhost:8000/departments/1?mode=reassign&reassign_to_department

## ❌ Как полностью удалить данные проекта

Если вы хотите полностью удалить проект с локальной машины, включая все данные базы данных, выполните следующую команду в корневом каталоге проекта.

    Внимание: Эта команда требует прав суперпользователя (sudo), так как файлы в каталоге database принадлежат пользователю root внутри контейнера.

bash
Копировать

sudo rm -rf database/

    Эта команда безвозвратно удалит папку database и все файлы внутри нее.

После этого можно удалять каталог с проектом.
