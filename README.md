# API Организационной Структуры (Python + FastAPI)

**Проект:** Реализация REST API для управления иерархической структурой организации (подразделения и сотрудники) в соответствии с принципами чистой архитектуры.

**Технологии:**
*   **Backend:** Python 3.14, FastAPI, SQLAlchemy (Async), Pydantic.
*   **База данных:** PostgreSQL.
*   **Инфраструктура:** Docker, Docker Compose, Alembic (миграции).

---

## 📝 О Проекте

Это учебный/демонстрационный проект, созданный для отработки навыков построения сложных, масштабируемых и поддерживаемых приложений на Python.

**Ключевые архитектурные решения, которые демонстрирует проект:**
1.  **Чистая Архитектура (Clean Architecture):** Четкое разделение на слои (API, Use Cases, Репозитории, Модели БД).
2.  **SOLID:** Принципы проектирования соблюдены на практике (например, через использование Generic-репозиториев и разделение ответственности).
3.  **Unit of Work (UoW):** Управление транзакциями базы данных в рамках одной бизнес-операции.
4.  **Обработка Ошибок:** Использование именованных доменных исключений и глобальных обработчиков в FastAPI.
5.  **Асинхронность:** Полностью асинхронный стек (ASGI, Async SQLAlchemy) для высокой производительности.

## 🚀 Как запустить проект

Весь проект упакован в Docker для простоты запуска и изоляции зависимостей.

### 1. Установите Docker и Docker Compose
Убедитесь, что у вас установлены:
*   [Docker Engine](https://docs.docker.com/engine/install/)
*   [Docker Compose](https://docs.docker.com/compose/install/) (обычно идет в комплекте с Docker Desktop)

### 2. Клонируйте репозиторий
```bash
git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git
cd YOUR_REPO
```

### 3. Соберите образы и запустите контейнеры
Эта команда скачает образы Python и PostgreSQL, установит зависимости и запустит базу данных.
```bash
docker compose up --build -d
```
*   `--build` — пересоберет образ приложения, если код изменился.
*   `-d` — запустит контейнеры в фоновом режиме (detached).

### 4. Примените миграции базы данных
После запуска контейнеров нужно создать таблицы в БД.
```bash
docker compose exec api alembic --config ./alembic/alembic.ini upgrade head
```
*(Если контейнер с приложением называется иначе, чем `api`, используйте его имя).*

### 5. Запустите сервер
Если вы остановили контейнеры после миграций, запустите их снова:
```bash
docker compose up --build -d
```

### 6. Проверьте работу API
Сервер будет доступен по адресу [http://localhost:8000/docs](http://localhost:8000/docs).
Откройте этот адрес в браузере — вы должны увидеть автоматически сгенерированную документацию Swagger UI (OpenAPI).

## 📋 Доступные Эндпоинты (из ТЗ)

Документация доступна по адресу [http://localhost:8000](http://localhost:8000) после запуска сервера.

### Подразделения (Departments)
*   `POST /departments/` — Создать подразделение.
*   `GET /departments/{id}` — Получить подразделение (с сотрудниками и поддеревом).
*   `PATCH /departments/{id}` — Переместить/переименовать подразделение.
*   `DELETE /departments/{id}` — Удалить подразделение (с поддержкой режимов `cascade` и `reassign`).

### Сотрудники (Employees)
*   `POST /departments/{id}/employees/` — Создать сотрудника в конкретном подразделении.

## 🧪 Что внутри (Структура проекта)

Проект следует слоистой архитектуре:
*   **app/api:** Слой API (FastAPI роутеры, эндпоинты). Не содержит бизнес-логики.
*   **app/core:** Ядро приложения (Unit of Work, базовые классы репозиториев, исключения).
*   **app/departments / app/employees:** Бизнес-логика (Use Cases) и доступ к данным (Репозитории) для конкретных сущностей.
*   **docker-compose.yml:** Описание сервисов (приложение и база данных).
*   **alembic/:** Папка с миграциями базы данных.
